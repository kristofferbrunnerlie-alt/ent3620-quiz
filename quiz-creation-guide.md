# ENT3620 Quiz Creation Guide
## Based on analysis of Mock Exams 1 & 2 (50 questions total)

### FORMAT
- 60% MC (4 options), 40% T/F
- ~20% use negative framing ("which is NOT", "would not be considered")
- Aim for 25 questions per quiz

### DIFFICULTY DISTRIBUTION
- 30% Easy (direct definition recall)
- 50% Medium (understanding, applying definitions to scenarios)
- 20% Hard (subtle distinctions, case-specific details, advanced concepts)

### TOPIC DISTRIBUTION (match exam weighting)
- Org structure: 16% (4 questions per 25)
- Adoption & diffusion: 16% (4 questions)
- IP protection: 14% (3-4 questions)
- Selecting/financing: 14% (3-4 questions)
- Innovation basics: 12% (3 questions)
- S-curves & tech evolution: 8% (2 questions)
- Cases (Moderna, Nokia, Metaverse): 10% (2-3 questions)
- Sources of innovation / R&D: 6% (1-2 questions)
- Other: 4% (1 question)

### DISTRACTOR DESIGN RULES
1. ALL wrong answers must be real course concepts used in wrong context (never random/silly options)
2. For MC: use same answer pool across multiple questions (e.g., AHP/real options/IRR/NPV)
3. For "NOT" questions: 3 correct items from the course + 1 that sounds plausible but isn't
4. Include "plausible outsiders" — terms that sound academic but aren't from this course
5. For T/F: change ONE word that flips the meaning (linear→nonlinear, different→shared)
6. Include scenario-based questions that require APPLYING a definition to a new situation

### T/F QUESTION RULES
- Statement must be precise enough that there's no ambiguity
- Use exact phrasing from textbook/lectures where possible
- Traps should test: wrong category (benefit listed as cost), wrong scope (international when it's national), wrong direction (more→less)
- Include some that are obviously true/false as confidence builders

### MC QUESTION RULES
- All 4 options should be similar in length and style
- Avoid "all of the above" (professor rarely uses it)
- Include distractors from adjacent concepts (e.g., "process development" when answer is "technological innovation")
- For case questions: include plausible-sounding details that are wrong

### REAL EXAM EXPECTATIONS
- The real exam will NOT reuse mock exam questions
- It WILL use the same difficulty level, language style, and framing
- Questions come from ALL mandatory readings (textbook chapters, articles, cases, lectures)
- The professor's "favorite" concepts (org structure, adopter categories, decision tools) will appear but in NEW formulations
- Must test understanding, not just recognition of memorized questions
- Create questions the student has NEVER seen before, testing the same concepts from new angles

### STUDENT-SPECIFIC WEAK AREAS (Kristoffer)
- Confuses "disruptive innovation" with "technological innovation" for accidental discoveries
- Gets tripped by negative framing ("NOT")
- Org structure vocabulary (organic/mechanistic, centralized/decentralized)
- IP details: working requirement, international copyright, R&D intensity by firm size
- Moderna case specifics (LNPs, born digital, 2-day vaccine design)
- Needs more scenario-based patent questions (what can/cannot be patented)
